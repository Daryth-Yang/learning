# 案例:Clawtics(龙虾)

> D5 公司的 AI 技能聚合工作台。

## 产品定位

类比国外:Adept、Rabbit R1、Notion AI 的演进方向
核心理念:把多个 AI 能力(写作、整理、分析)聚合到一个工作台

## 三大功能分类

1. **帮我整理**:把散落的原始材料变成结构化资产
   - 会议纪要、URL/视频/播客摘要、多表合并对账、拍照扫描数字化、论文 PDF 入库、行业动态抓取

2. **与我探讨**:丢一份长文档和它来回对话
   - 合同/财报/政策深读、多文件交叉对比

3. **为我产出**:把想法写成能交付的稿子
   - 撰写文稿(邮件/周报/PRD/博客)、PPT 生成、高质量翻译、演讲稿/口播稿、简历优化

## 教练用法

### 模块 1 W6(Agent / Workflow)
让 Daryth 去 Clawtics 体验 5 个不同的 skill,观察并分析:
- 每个 skill 的输入是什么?输出是什么?
- 它内部可能用了什么技术(prompt? RAG? Tool use?)
- UI 上是怎么帮用户管理多个 skill 的?

### 模块 2 W3(对话式 UX)
深度拆解 Clawtics 的 UX:
- 它怎么帮用户发现合适的 skill?
- 它的"工作台 / Clawtics / 技能"3 个 Tab 是怎么分工的?
- 有没有"上一步可撤销"等容错设计?

### 模块 4(独立 MVP)
可作为 Daryth 自己 MVP 的灵感来源(但不要抄,做差异化)
