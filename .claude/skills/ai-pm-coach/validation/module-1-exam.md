### Part 2:开放问答(10 题,共 50 分)

教练随机抽 10 题,Daryth 现场口述/打字回答。每题 5 分。

#### 题库(教练每次抽不同的 10 道)

1. 用 200 字解释什么是 Token,为什么 Token 决定了 API 成本。
2. 客户问"为什么我们的 RAG 系统有时候答非所问",你怎么排查?列至少 5 个可能原因。
3. 给一个非技术老板解释"什么是 Agent",用 3 分钟。
4. Prompt 工程、RAG、Fine-tuning 三种方式有什么区别?各自最适合什么场景?
5. 如果让你给"律师助手"产品做选型,你最看重哪 3 个维度?为什么?
6. 什么是 Prompt 注入?它和越狱有什么区别?
7. 你怎么评测一个 RAG 系统的好坏?列出至少 3 个指标。
8. 多模态模型和单模态模型的核心区别在哪?给 2 个多模态产品例子。
9. Temperature 设置成 0 和设置成 1.5 的输出会有什么区别?什么场景用什么值?
10. MCP 是什么?为什么 Anthropic 要推它?它跟 Function Calling 什么关系?
11. 你的公司想做"AI 自动写周报"功能,你作为 PM 怎么拆解需求?
12. 上下文窗口是什么?为什么有的模型卖点是"1M token 上下文"?
13. 解释什么是"幻觉",举一个真实案例,然后说怎么减少幻觉。
14. 为什么有些公司要做"私有化部署"或者"本地模型"?
15. Function Calling 怎么工作的?用一个例子说明流程。
16. 你怎么决定一个 AI 产品该用 Claude 还是 GPT 还是国内模型?
17. RAG 系统中"重排(Reranking)"是什么?为什么需要它?
18. 数据飞轮是什么?给一个 AI 产品的数据飞轮例子。
19. 流式输出和非流式输出对用户体验有什么影响?什么场景必须流式?
20. 如果让你给一个 AI 产品控制成本,你会从哪些方面入手?(列至少 4 个杠杆)

#### 评分维度(每题)

- ✅ 答对核心(3 分)
- ➕ 举出实例(1 分)
- ➕ 涉及权衡和深度(1 分)

### Part 3:实战案例分析(30 分)

教练给一个伪需求,Daryth 用 30-60 分钟做需求拆解。

#### 案例题库(教练随机选 1)

**案例 A**:某教育机构想做"AI 老师"产品,自动给中学生答疑解惑。
- 拆解:核心 AI 能力 / 选型 / 风险 / 评测 / 商业模式

**案例 B**:某律所想给内部律师做"合同智能审查"工具。
- 拆解:RAG vs 微调 / 数据安全 / UX 设计 / 上线评测

**案例 C**:某 SaaS 公司想给自家工单系统加"自动回复"能力。
- 拆解:技术方案 / 召回率/精确率 trade-off / 成本测算 / 兜底设计

**案例 D**:某直播平台想做"AI 数字人主播",取代部分人工主播。
- 拆解:多模态技术栈 / 数据准备 / 风险(版权/审核)/ 商业账

#### 评分维度

- 业务理解(8 分):有没有看懂需求里的痛点
- 技术方案(10 分):选型合不合理,有没有讲清三种方式的对比
- 风险识别(6 分):有没有识别幻觉/安全/成本/合规等关键风险
- 表达逻辑(6 分):结构清晰、有图有数据更好

## 教练验收行为指南

1. **不要放水**:Daryth 想拿 offer,放水等于害她。该不及格就不及格。
2. **要鼓励**:不及格也要肯定她做对的部分,然后清晰指出薄弱点和复习路径。
3. **要具体**:反馈不能"逻辑要更清晰",而是"在 W4 报告里你没考虑成本因素,请重写第 3 节"。
4. **结果写入 progress.md**:把分数和薄弱点详细记下来,作为后面模块的参考。
