# 模块 4:实战与作品集(3-4 周)🎯 决定能否拿 offer

**周期**:3-4 周 | **目标**:整合所有学习产出,完成可投递的求职包

> 注:模块 4 详细内容在 Daryth 接近模块 3 完成时由教练扩写完整。这里先放骨架。

---

## 模块整体目标

- 完成 3 篇深度 AI 产品拆解报告(每篇 2000+ 字)
- 独立从 0 做出 1 个可访问的 AI 产品 MVP
- 完整的简历 + 作品集页面 + 面试准备
- 至少完成 3 次模拟面试

## 周次概览

| Week | 主题 | 输出 |
|---|---|---|
| W1 | 深度拆解 3 个标杆 AI 产品 | 3 篇 2000 字拆解报告 |
| W2 | 独立做 AI 产品(开始) | 立项 + PRD + 设计稿 + 开始用 vibe coding 实现 |
| W3 | 独立做 AI 产品(完成) | MVP 上线 + 公开访问链接 |
| W4 | 求职包完善 | 简历 + 作品集页面 + 模拟面试 |

## 求职包应包含

- 1 份针对 AI PM 的简历(突出项目经验,而不是 UI 履历)
- 作品集网站(可以用 Notion / 个人网站 / GitHub Pages)
- 包含:产品拆解 / 选型报告 / PRD 案例 / 独立项目 / 学习记录

## 模拟面试题型(教练届时出题)

- 行为面:为什么从 UI 转 PM
- 业务理解面:给一个产品做 0-1 拆解
- 技术理解面:解释 RAG / Agent 工作原理
- 案例面:给一个伪需求做完整方案
