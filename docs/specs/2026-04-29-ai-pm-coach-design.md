## 1. 背景与目标

### 1.1 学习者画像

- **当前身份**: 某 AI 初创公司 UI 设计师
- **设计经验**: C 端 App、B 端平台、官网设计(经验广泛)
- **AI 基础**: Level 2 — 了解 prompt / agent / RAG / API 等概念,但不深入
- **编码能力**: 仅做过 vibe coding(看不懂代码)
- **英语**: 简单技术英文 OK,长文吃力
- **学习风格**: 主要是视频学习,其他形式可接受
- **每日投入**: 2-4 小时 / 天 (约 14-28 小时/周)

### 1.2 公司产品组合(关键学习资源)

学习者所在公司的 4 个产品恰好覆盖 AI 行业最主流的 4 种产品形态,将作为贯穿全程的实战案例:

| 产品 | AI 产品类型 | 关联学习模块 |
|---|---|---|
| **Clawtics(龙虾)** | AI 技能聚合工作台 | M1-W6 Agent、M2 工作流设计 |
| **数字人 / 视频生成** | 多模态生成式 AI | M1-W2 多模态、M2 产品设计 |
| **获客产品** | 营销/数据驱动 AI | M3 数据分析、M3 商业化 |
| **律师助手** | 垂直 RAG | M1-W4/5 RAG、M1-W7 安全 |

### 1.3 目标定位

**做能独立带项目、懂业务、懂 AI 技术原理的 AI 产品经理**,方向以 A(AI 应用 PM)+ 部分 B(AI 功能 PM)为主。

### 1.4 三重目标(并行达成)

- **B**:获得 AI PM offer / 内部转岗
- **C**:系统补齐能力体系
- **D**:能独立产出 AI 产品(独立开发者方向)

### 1.5 学习者最关注的弱项

按强化优先级排序:

1. 看不懂代码(担心永远是门外汉)
2. 不懂技术原理(怕跟工程师对话被识破)
3. PRD 文档写不好
4. 缺业务/商业思维
5. 英语阅读累

## 2. 整体路径(4-5 个月,4 模块)

```
模块 1:AI 技术地基 (8 周)        ← 基础打牢,深度版
模块 2:AI 产品设计 (4-5 周)
模块 3:PM 通用能力 (4-5 周)      ← 最大短板
模块 4:实战与作品集 (3-4 周)      ← 决定能否拿 offer

总周期:约 19-22 周(4.5-5.5 个月)
```

每个模块都包含:**理论学习(20%)+ 产品拆解(30%)+ 实战项目(40%)+ 验收(10%)**

## 3. 模块 1:AI 技术地基(8 周,深度版)

> 目标:能跟工程师对话不被忽悠,理解技术原理,能看懂代码做了什么

| 周 | 主题 | 核心要点 | 周作业 |
|---|---|---|---|
| W1 | LLM 工作原理 | 预训练 + RLHF / Token / 上下文 / Temperature / 流式输出 | 同一 prompt 跑 3 个 temperature,写 800 字对比分析 |
| W2 | 模型生态 + 多模态 | Claude/GPT/国内模型矩阵 / 多模态(图/视频/语音)/ 大小模型选择 | 做《主流模型对比矩阵》+ 给公司 4 个产品各推荐模型并说理由 |
| W3 | Prompt 工程系统化 | 结构化 Prompt / CoT / ReAct / Prompt Caching / 结构化输出(JSON Schema)| 挑工作场景,Prompt 迭代 5 轮日志 |
| W4 | **三种"教 AI"方式对比** ⭐ | Prompt vs RAG vs 微调 / 场景选择 / 成本-效果对比 | 给"律师助手"写技术方案选型报告 |
| W5 | RAG 深入 + 知识库 | 向量 / 嵌入 / 检索全链路 / 失败模式 / 评估方法 | 在 Coze/Dify 搭知识库 bot,测 20 题打分 |
| W6 | Agent + Function Calling + MCP | Agent 三件套(Memory/Planning/Reflection)/ 单/多 Agent / Workflow / FC 原理 / MCP 标准 | 设计一个 Agent 能力框架(对应公司一个产品)|
| W7 | 评测 + 安全 + 数据飞轮 | Benchmark / 业务评测 / 在线评测 / 幻觉 / 越狱 / Prompt 注入 / 反馈闭环 / 成本管理 | 给公司一个产品做"评测方案 + 安全方案" |
| W8 | 代码可读性 + API 实战 + 验收 | Python 基础语法 / 目录结构 / JSON-YAML / API 调用 / 错误日志 | 让 AI 给你之前 vibe coding 项目逐行讲解 + 模块验收 |

### 模块 1 验收标准

- 10 道开放问答(技术原理 + 产品视角混合)
- 1 个真实案例分析
- 8 周作业完整入仓库
- 评分 ≥ 70 分通过,否则薄弱模块复习重考

## 10. Skill 运作机制

### 10.1 启动方式

学习者通过以下任一方式激活 skill:

- 显式调用:"激活 AI PM 教练" / "我要继续学习" / 直接调用 skill
- 任何模糊触发:"今天学什么?" / "这周该做什么?"

### 10.2 每次会话流程

1. 读 `progress.md` → 知道学习者当前在哪一周哪一天
2. 检查上次作业完成情况(若有)
3. 给出今天/本周具体任务(2-4 小时可完成)
4. 验收上次作业(若提交)
5. 更新 `progress.md`
6. 必要时扩充课程 / 资源

### 10.3 验收机制

每个模块结束:

- 出 10 道开放题让学习者作答
- 出 1 个真实案例让学习者做需求/技术分析
- 评分:从「技术理解」「产品思维」「逻辑表达」三维度打分
- 不通过(<70 分):薄弱模块重学,7 天后重考
- 通过:进入下一模块,作业归档到 `portfolio/`

### 10.4 资源策略

- 中文资源优先(B 站、公众号、知乎)
- 必读英文资源:用 AI 翻译成中文版,关键术语保留英文对照
- 视频资源占比目标:50% 视频 + 30% 文章 + 20% 实操

### 10.5 公司产品作为案例

模块 1 W4、W7,模块 2-4 的实战案例**优先使用学习者公司的产品**作为分析对象:

- 律师助手 → 安全/RAG/合规专题
- 数字人 → 多模态/UX/成本专题
- 获客产品 → 数据/商业化专题
- Clawtics → Agent/工作流/产品架构专题

附带好处:作业可能直接对工作有用,可向公司争取更深的项目机会。

## 11. Skill 持续更新机制

skill 不是一次性产物,在使用中持续演进:

- **学习者反馈触发更新**:"这周内容太浅 / 太深 / 看不懂" → 调整课程
- **新学习需求触发扩展**:遇到新概念时,在 resources.md 添加专题
- **行业新进展触发刷新**:出现重要新技术(如新模型、新协议)时,补入对应周
- **作品集成长触发结构调整**:portfolio 内容多了之后,重新组织分类

每次重要更新会在 `SKILL.md` 顶部记录变更日志。

## 13. 风险与应对

| 风险 | 应对 |
|---|---|
| 中途松懈 | 副线任务保持节奏 + 每周复盘 |
| 工作太忙没时间 | 进入"低强度模式":每天最少 30 分钟,只完成核心作业 |
| 某个模块卡住 | 暂停新学习,重做该模块,直到验收通过 |
| 公司产品突然变化 | 案例库随时更新 |
| 学习内容跟不上行业变化 | 每月扫描新动向,必要时插入"专题周" |
